Introduction Last updated: 2023-02-04

The InExTracker Api is a RESTful Api that provides a secure and easy way to interact with the inextracker app. This api was built using Django-rest-framework and allows you to perform various operations on the inextracker data without usig the web interface.

With this api, you can perform the following operations.

  • Retrieve data on income and expenses
  • Create new entries for income and expenses
  • Edit existing entries for income and expenses
  • Delete existing entries for income and expenses
  • Retrieve and manipulate data for different users

Authentication

Using the inextracker api to make api calls requires authentication which must be added to the header.

Get Token

to get the authentication token, you have to be registered first as a user, then you can request for an api key.

Retrieve Token

You can always retrieve your inextracker api token if forgotten by simply making a get request with the following code

GET / http://localhost/api/get-token

you need to pass in your username and password in the parameters

{
 "username":"your_username",
 "password":"your_password"
}
and on success, 
{
	"token": "e82c490f44603ed8428092e84ac62198248b68c6"
}

Requests

Making use of the inextracker APi requires header authentication. your authentication token must be passed in the header and not in the parameters. . The available requests methods are

  • GET
  • POST
  • PUT
  • DELETE

GET Requests

Get requests are available for income and expenses and should be made to the following urls

GET Income

the endpoint for making a get request for all income related to a user is 

GET / localhost:8000/api/income

The a sample response from the code above is given below 

[
  {
	"id": "1",
	"amount": 20000,
	"category": travel,
	"description": text ...,
  },
  {
	"id": "2",
	"amount": 40000,
	"category": rent,
	"description": text ...,
  },

]

GET Expense

the endpoint for making a get request for all expense related to a user is
GET / localhost:8000/api/expense
The a sample response from the code above is given below 

[
  {
	"id": "1",
	"amount": 20000,
	"source": investments,
	"description": text ...,
  },
  {
	"id": "2",
	"amount": 40000,
	"source": salary,
	"description": text ...,
  },
]

POST Requests

making a post request for income and expense are simiar. all fields are required except for the id which is automatically generated

the endpoint for making an income post request is
POST / localhost:8000/api/income

the endpoint for making an expense post request is
POST / localhost:8000/api/expense
A sample response from the code above is given below 

{
	"success": "object has been created successfully",
 }

Delete Requests

making a delete request for income and expense are simiar. you have to specify the id of the object you want to update

the endpoint for making an income delete request is
POST / localhost:8000/api/income/{id}/delete

the endpoint for making an expense delete request is
POST / localhost:8000/api/expense/{id}/delete
A sample response from the code above is given below

On Success
{
	"success": "object has been deleted successfully"
}
Error
{
	"error": "object does not exist",
}
